# 20-03 — Ambient effects: BlackHole / Dust / Signals / Race-specific

## Goal

Вынести 7-9 self-contained `setupXxx` методов ambient-эффектов в отдельные модули `src/game/scenes/starmap/ambient/`.

## Что выносим

### Universal ambient (для всей карты)

| Метод | Размер | Назначение |
|---|---|---|
| `setupBlackHole` | ~60 строк | Чёрная дыра в центре |
| `setupHomeOrbiter` | ~50 строк | Орбитальные элементы около HOME |
| `setupCosmicDust` | ~35 строк | Космическая пыль |
| `setupSignalPulses` | ~50 строк | Сигнальные импульсы |
| `setupRandomSignals` | ~45 строк | Случайные signal-выбросы |

### Race-specific (для уникальных планет)

| Метод | Размер | Назначение |
|---|---|---|
| `setupTorRing` | ~20 строк | Кольцо вокруг TOR |
| `setupVeranLightning` | ~40 строк | Молнии вокруг VERAN |
| `setupRelictMourning` | ~25 строк | Меланхоличный эффект на RELICT |

## Pattern

Каждый эффект → файл `src/game/scenes/starmap/ambient/<name>.ts`:

```typescript
// src/game/scenes/starmap/ambient/blackHole.ts
import type Phaser from 'phaser'

export function setupBlackHole(scene: Phaser.Scene): void {
  // тело из StarMapScene.ts setupBlackHole
  // `this.X` → `scene.X`
  // `this.add.graphics()` → `scene.add.graphics()`
}
```

Если эффект нужно teardown'ить — возвращаем cleanup-функцию:

```typescript
export function setupBlackHole(scene: Phaser.Scene): () => void {
  // setup logic
  return () => { /* cleanup */ }
}
```

В `StarMapScene.create()`:

```typescript
import { setupBlackHole } from './starmap/ambient/blackHole'
import { setupCosmicDust } from './starmap/ambient/cosmicDust'
// ...

create() {
  // ...
  setupBlackHole(this)
  setupHomeOrbiter(this)
  setupCosmicDust(this)
  setupSignalPulses(this)
  setupRandomSignals(this)
  setupTorRing(this)
  setupVeranLightning(this)
  setupRelictMourning(this)
}
```

## Tasks

1. Создать папку `src/game/scenes/starmap/ambient/`
2. Для каждого setup-метода:
   - Создать файл `<name>.ts` в `ambient/`
   - Скопировать тело метода
   - Заменить `this.X` на `scene.X`
   - Если используются константы из starmap (DPR, etc.) — импортировать
3. В `StarMapScene.ts`:
   - Импортировать новые setup-функции
   - В `create()` вызывать `setupX(this)` вместо `this.setupX()`
   - Удалить method-тела
4. После каждого extract'а: `npx tsc --noEmit`
5. После всех: ручная QA — открыть starmap, проверить:
   - Чёрная дыра в центре анимирует
   - Космическая пыль летает
   - Сигнальные импульсы появляются
   - Кольцо TOR / молнии VERAN / RELICT эффект — на местах
6. Commit: `refactor(starmap): extract ambient effects (20-03)`

## Risk notes

- Низкий риск — каждый эффект изолирован, не зависит от других
- Главное: некоторые методы хранят references на созданные объекты в `this.someField` для последующего обновления. Если такое — нужно либо вернуть references из функции, либо оставить хранилище в scene и передавать туда
