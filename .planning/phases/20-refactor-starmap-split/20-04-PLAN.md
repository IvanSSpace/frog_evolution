# 20-04 — Bridges: Starfield / Popovers / Ship / HUD

## Goal

Вынести 4 более сложных модуля которые имеют bridge-логику между Phaser и React/eventBus/store.

## Что выносим

### 1. Starfield → `src/game/scenes/starmap/starfield.ts` (~200 строк)

Методы: `setupStarfield`, `drawLines`, `redrawMainLines`, `buildBgBatch`, `renderSystem`, `createSparkleAt`

Это базовый рендер карты. Все методы без cross-зависимостей с camera/popover.

### 2. Popovers → `src/game/scenes/starmap/popovers.ts` (~600 строк)

Методы:
- `openPhaserPopover`, `closePhaserPopover`
- `openBgNamePopup`, `closeBgNamePopup`, `scheduleBgNamePopup`
- `makePhaserButton`
- `selectSystem`
- `handlePlanetPress`
- `playUniqueAnimation`, `runAnimComponent`, `getAnimationDurationMs`

⚠️ Popover'ы emit'ят `eventBus`, держат refs на DOM-элементы, реагируют на camera move.

Стратегия: создать класс `PopoverController` с `(scene: StarMapScene)` constructor'ом.

### 3. Ship Controller → `src/game/scenes/starmap/shipController.ts` (~150 строк)

Методы: `setupShipSprite`, `teardownShipSprite`, `applyShipState`

Уже почти изолирован — `ShipSprite` это отдельный класс. Нужно собрать остатки в один файл с lifecycle-management.

⚠️ Имеет Zustand subscription и eventBus listener. При teardown — must unsubscribe.

### 4. Coordinates HUD → `src/game/scenes/starmap/coordinatesHUD.ts` (~130 строк)

Метод: `setupCoordinatesHUD`

Хранит smoothed FPS history, обновляет публичные `hudFps/hudVisible/hudTotal` поля сцены через `scene.events('update')`.

Стратегия: класс `CoordinatesHUDController` с публичными readable getter'ами.

## Pattern: Controller class

Для модулей с lifecycle — класс контроллера:

```typescript
// src/game/scenes/starmap/shipController.ts
import Phaser from 'phaser'
import type { StarMapScene } from '../StarMapScene'
import { ShipSprite } from '../../effects/ShipSprite'
import { useGameStore } from '../../../store/gameStore'

export class ShipController {
  private shipSprite: ShipSprite | null = null
  private unsubscribe: (() => void) | null = null
  private lastStateSig = ''

  constructor(private scene: StarMapScene) {}

  setup(): void {
    // содержимое setupShipSprite
  }

  teardown(): void {
    // содержимое teardownShipSprite
  }

  private applyState(ship: ShipState | null): void {
    // содержимое applyShipState
  }
}
```

В `StarMapScene`:

```typescript
private shipController!: ShipController

create() {
  // ...
  this.shipController = new ShipController(this)
  this.shipController.setup()
  this.events.once(Phaser.Scenes.Events.SHUTDOWN, () => this.shipController.teardown())
}
```

## Risk notes

- **Popovers** — самый сложный модуль. DOM-refs, eventBus, RAF loops — много мест для ошибок. Делать аккуратно, можно разбить на 2 коммита (capital popovers + bg popups).
- **ShipController** — уже почти изолирован, риск низкий
- **HUD** — публичные поля сцены `hudFps/hudVisible/hudTotal` — если App.tsx читает их через `getStarMapHUD()`, нужно сохранить тот же API
